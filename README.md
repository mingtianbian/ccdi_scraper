# CCDI 官员审查调查数据抓取系统

## 项目简介
本项目是一个针对中央纪委国家监委网站（ccdi.gov.cn）“审查调查”频道的专业级数据采集工具。系统具备**环境自愈**、**反爬虫对抗**、**数据结构化**、**断点续传**及**并发采集**等核心能力，能够自动化获取并解析中管干部、省管干部等各类人员的违纪通报信息。

## 核心功能

### 1. 零配置启动 (Start Script)
- **环境自愈**：通过 `start.py` 自动检测并安装 Python 依赖。
- **内核集成**：自动下载 Chromium 浏览器内核，无需手动配置 WebDriver。

### 2. 高级仿真与反爬 (Anti-Detection)
- **Playwright 驱动**：使用真实浏览器内核，而非简单的 HTTP 请求。
- **特征屏蔽**：自动注入脚本抹除自动化测试痕迹（如 `navigator.webdriver`）。
- **人机模拟**：随机等待时间和鼠标行为模拟，降低封禁风险。

### 3. 数据结构化 (Structured Data)
- **智能解析**：自动提取标题、来源、发布时间、正文内容。
- **姓名/职位分离**：内置算法尝试从标题中分离“人员姓名”和“原任职位”。
- **Excel 导出**：数据最终保存为标准 `.xlsx` 格式，方便查阅。

### 4. 高效并发与续传 (Concurrency & Resume)
- **并发采集**：**同时开启 6 个线程**，并行抓取所有分类（中管干部/省管干部 x 执纪审查/党纪处分）。
- **断点续传**：内置 SQLite 数据库 (`data/ccdi.db`) 实时保存进度。
    - 如果程序中断，**重启即继续**。
    - 自动跳过已采集的 URL，避免重复工作。

### 5. 可视化界面 (CLI)
- **Rich 终端 UI**：提供专业的进度条、状态表和彩色日志。
- **实时监控**：不仅看日志，更能通过进度条直观看到每个分类的抓取进度。

## 安装与使用

### 1. 运行环境
- Windows 系统
- 建议预装 Python 3.8+ (如果未安装，脚本可能会尝试调用系统 python)

### 2. 启动方式
在项目根目录下，双击运行或在终端执行：

```powershell
python start.py
```

*首次运行时，程序会自动创建虚拟环境、安装依赖库并下载浏览器内核，这可能需要几分钟时间，请耐心等待。*

### 3. 数据输出
- 采集过程中的数据实时存储在 `data/ccdi.db`。
- 程序运行结束后，会自动将数据库内容导出为 `.xlsx` Excel 文件，保存在 `data/` 目录下，文件名包含时间戳。

## 常见问题

**Q: 为什么启动时下载很慢？**
A: 首次启动需要下载 Playwright 的浏览器内核（约 100MB+）。取决于网络环境。

**Q: 程序中断了怎么办？**
A: 直接重新运行 `python start.py` 即可。程序会检查数据库，跳过已经下载过的文章，继续下载剩余部分。

**Q: 如何修改并发数量或间隔时间？**
A: 修改 `src/config.py` 文件中的配置项。
